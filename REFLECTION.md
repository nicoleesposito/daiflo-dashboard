1. For my midterm assignment, I created an application called Daiflo, which is a personal productivity dashboard that brings essential daily tools into one place. Users can check the weather, manage a to-do list, read the latest news, and write notes all from a single interface. For this assignment specifically, I added an AI Summarization Feature to the notes widget. When users press the Summarize button, the OpenAI API is called and summarizes the note. The Wikipedia REST API is also used to add additional context for a specific topic found in the note. I built this feature into the platform because, with Daiflo being a productivity board, I expect users to write notes during lectures or work meetings. The summarize button makes it easy to get a quick version of what was written without reading an extensive note, helping users quickly recall information at a later time.

2. Something that surprised me while building this feature was how streamlined the process was when connecting a cloud API into the code. It felt familiar to how I was previously incorporating API keys, and not much additional setup was required. However, I had a lot of difficulty getting the API to successfully generate a response. I started with Google AI Studio and the Gemini API. Although connecting it was easy, the platform would constantly say I exceeded my usage despite being on the free tier. After troubleshooting this several times with Claude, I decided to add funds to my account, but ran into another roadblock since Google requires a $25 minimum. I ended up switching to OpenAI, which only required a $5 minimum and was more straightforward to set up. If I had gone with OpenAI from the start, the process would have matched my previous experience with connecting APIs exactly.

3. Working with a cloud API taught me that $5 goes a long way. Generating a summary barely costs me a fraction of a cent per request, meaning I have a long way to go before my balance runs out. I also learned that there are several other features available beyond text generation, such as image generation, which I am excited to explore in future projects. Claude did a good job helping me find the root cause of the issue with Google's API, but since it wasn't financially reasonable to meet the $25 minimum, switching to OpenAI was my next option. It was also easier to use overall since Google AI Studio kept lagging on my computer, while OpenAI performed better.

4. Something I would do differently next time is research how each AI company handles free tier access before committing to one. Google advertises a free tier, but in practice, it still requires billing to be connected, which wasn't made clear upfront. If I had another week, I would explore adding a summarize button to the RSS news feed, which would require restructuring the layout but would be a natural extension of what I built here.
